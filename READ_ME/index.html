My Sample App

This download package contains an Android Studio project for a fully functioning Mobile App, which uses your configured AWS resources and identifiers, to demonstrate the project features you've selected. You can use this project as a starting point for your own product development, or you can follow the instructions below to copy and paste the required code into your own product.

IMPORTANT: This package was custom generated for you, using your selected project options. It contains your own resource and account identifiers and should be kept private.

  • Setup
    • Integrated Development Environment

      • 1. System Requirements


        Your system must include the following:
          - Android Studio 1.4 or newer
          - Android SDK 4.4 (KitKat) API Level 19 or newer
          - Android SDK Build-tools 23.0.1
          - Android Device with Android OS 4.0.3 (IceCreamSandwich) API Level 15 or newer

      • 2. Install Android Studio


        If you have not already installed the Android Studio IDE (Integrated Development Environment) on your host, you can download it from the following location:
        https://developer.android.com/sdk


      • 3. Install Android SDK (Software Development Kit)


        If you have not already installed the latest Android SDK, you can do so within Android Studio.

        Go to...

        Tools -> Android -> SDK Manager -> Appearance & Behavior -> System Settings -> Android SDK


        Then, select the Android SDK Platform version you would like to use. The SDK Manager will download the appropriate Android SDK Platform version and apply any available updates.

      • 4. Install AVD (Android Virtual Device)


        If you have not already installed an Android Virtual Device, you can do so within Android Studio.

        Go to...

        Tools -> Android -> AVD Manager


        Then, click "+ Create Virtual Device" and select the device you would like to use in the Android emulator (e.g., Nexus 5). Select the appropriate image from the list (e.g., Lollipop API Level 22 x86 Android 5.1.1) and click the "Next" button. Name the AVD and click the "Finish" button.

  • Compile and Run
    • Launch App

      • 5. Import MySampleApp Project into Android Studio


        Once you have downloaded and unzipped the sample app project files, please open Android Studio and choose...

        Import project (Eclipse ADT, Gradle, etc.)


        Find the folder where you extracted the zip file contents and select the "MySampleApp" sub-folder. Android Studio will import the project and build it using Gradle.

        Note that the project may have several dependencies, which Gradle will automatically retrieve from online repositories. The first time you build the app, it may take longer than subsequent incremental builds.

      • 6. Run MySampleApp in Debug Mode


        Once Gradle finishes building, you can click the "Debug app" icon to launch the app in the Android emulator.

    • User Identity - Try It!

      • 7. Demonstrate User Identity


        Launch MySampleApp in the Android emulator. If you've selected to enable User Sign-in in your project, you should sign in to the app. Then, click on the "User Sign-In" demonstration section and then click the "Demo User Identity" button. You will see a unique identifier for the user. This identifies the user to AWS systems and provides the user temporary permissions to access your configured AWS resources.

      • 8. Verify User Identity


        Go to the AWS Cognito console, and you should see the count of users in your Cognito Identity Pool is now greater than zero.
        https://console.aws.amazon.com/cognito/


    • Push Notifications - Try It!

      • 9. Demonstrate Push Notifications


        Launch MySampleApp in the Android emulator and click on the "Push Notifications" demonstration section of the app. Click on the "Demo Push Notifications" button at the bottom of the screen, and you will see a checkbox "Receive Push Notifications", which lets you enable or disable push notifications feature on the app. Once enabled, the app will create an endpoint with the app's GCM device token with Amazon SNS. Then it will automatically subscribe the app to a default SNS topic ("ALLDEVICES"). You can toggle topic subscription by clicking on a topic in the list. In order to run this on a device, it must have Google API enabled.

      • 10. Verify Push Notifications


        Go to the Amazon SNS console, click on "topics" in the left menu, and select a topic to which you want to publish a message. Compose in plain text (RAW) in the next screen and click "Publish". If the app enables "Receive Push Notifications" and is subscribed to the given topic, it will receive the message. If the app is running in the foreground, the message will be shown in a dialog box in the app. In the app is not in the foreground, the message will be shown in the notification center.
        https://console.aws.amazon.com/sns/


    • User Profile Data - Try It!

      • 11. Demonstrate User Profile Data


        Launch MySampleApp in the Android emulator and click on the "User Data Storage" section of the app. Then click "Demo User Profile Data" button in the next page. Once the "User Profile Data" page is loaded, it loads the profile data for the current user and it updates the appearance with the user's theme. You will see four buttons which allow you to update the theme for the current user. Try changing the theme for the current user by tapping on one of the buttons.
        In order to sync user profile data across multiple devices, the user must sign in using an identity provider, such as Facebook or Google.

      • 12. Verify User Profile Data


        Launch the sample app in the Android emulator, log-in to one of the identity providers, such as Facebook or Google. Go to the "User Profile Data" demonstration page and pick a different color theme. The color theme is saved to the user's profile data. Now to verify that it was saved correctly, install the "MySampleApp" on a different device or run it on a different Simulator than the one you previously used. Sign-in with the same user and you should see the color theme change to the one you previously selected.

    • User Files - Try It!

      • 13. Demonstrate User File Storage


        Launch MySampleApp in the Android emulator and click on the "User Data Storage" section of the app and then click "Demo User Files" in the next page. You will see two folders "Public" and "Private". The first folder lets you see and manage files that are public to all app users. The second folder is only available to signed-in users and you will be promoted if you aren't. It allows you to manage files that are private where no other users have access to them. Both folders share the same user interface. Let's use Public files as an example.

        After clicking on the folder, please wait for the app to load the list of files from the Amazon S3 bucket. The path at the top tells you where you are in the bucket.

        The next section is the on-device cache summary. The columns, from left to right, are maximum size of disk space for local disk cache, how much is used for cache, how much is left, and the size of pinned content which is cached but doesn't count towards the cache limit you set. You can change the cache size or clear cache via the menu on the top right corner. The bottom part is a list of files in the Amazon S3 bucket, where items in black are downloadable objects with its size on the next line and those in blue are folders which you can navigate into. By clicking on a downloadable object, you can choose from "Open Remote URL", "Download", "Download & Pin", and "Delete Remote File". You will see download progress next to the object if you choose download, and you will have the option to open or delete it once download finishes. If the local cache is full, the oldest object in the cache will be deleted in order to make room for new downloads. You can prevent an object from being deleted by pinning it. To upload an image, click "Upload" from the menu. Upload will start once you choose a file. In order to group files, you can create folders using the "New Folder" from the menu.



      • 14. Verify User File Storage


        Go to Amazon S3 console. If you upload a new image file to S3, remove an existing file, or create a new folder inside the app, you will see it reflected in the S3 console as well. Likewise, any changes you make via the S3 console will be reflected in the app next time you load the "Demo User Files" section.
        https://console.aws.amazon.com/s3


    • NoSQL Database - Try It!

      • 15. Demonstrate NoSQL Database


        Launch MySampleApp in the Android emulator and click on the "NoSQL Database" section of the app and then click "Demo NoSQL Database" at the bottom of the next screen. You will see a list of the DynamoDB tables that were added to your project. Tap on the table that you would like to use in the demo.

        A list of example operations to view data in your table will now be shown. Tap the "Insert Sample Data" button in the bottom left of the screen to add random sample data to your table. Click on any of the list items to demonstrate running get, query, and scan operations on your table. When you are done, click on "Remove Sample Data" in the bottom right to remove the data that was inserted by the demo.



      • 16. Verify Data in DynamoDB Console


        Go to the Amazon DynamoDB console. You can click the table name in the "Resources" panel within Mobile Hub to navigate directly to your table in the Amazon DynamoDB console. Once you have inserted sample data into one of your tables, you will see it reflected in the DynamoDB console under the "items" tab for your table. Also, any entries you add to your table from the console will be reflected in the app when you use the Scan without Filter operation in the sample app.

23. User File Manager - Transfers Public and Private Files

        

This section simply provides information about the generated code in the MySampleApp project.

        

The User File Manager uploads and downloads files from Amazon S3. It caches downloaded files locally on the device in a size-limited cache. Downloaded files may be pinned to the cache, so that they are not automatically removed when the cache size limit is exceeded. The User File Manager provides access to two folders in the Amazon S3 bucket, one called "public/" for public files, which are accessible to any user of the app, and one called "private/" which contains a sub-folder for each Amazon Cognito identified user. Files in the user's private folder can only be accessed by that user. The User File Manager serves as the application's interface into the file-related functionality of the User Data Storage feature.

        
MySampleApp/app/src/main/java/com/amazonaws/mobile/content/UserFileManager.java
